--- layout: post title: "Pig-Min 스킨의 (간단한) 변경점." date: "2008-03-18 02:23:23+09:00" categories: "공지사항" redirect_from: /tt/1489 --- 이 글에 comkid 님과 애독자 님이 단 리플에 의거해, Pig-Min 스킨을 소폭 변경했습니다.

- 검색 결과 / 카데고리 클릭 / 태그 클릭 등의 목록이 나올 때, 글은 출력되지 않음.
  -> 과거 글까지 나와 한 페이지에 출력되는 텍스트가 너무 많았음.
  -> 다음 페이지로 넘어가기 위한 인덱스는 글의 끝에 있어, 내려가 클릭하기 불편했음.

- 검색 결과 / 카데고리 클릭 / 태그 클릭 등의 목록이, 1 페이지에 10개까지 나옴.
  -> 과거 5개라, 검색 결과가 많을 시 페이지를 넘기며 보기 불편했음.

이게 더 편한지 과거의 방식이 더 편했는지, 피드백 남겨주시면 감사드리겠습니다. 목록을 찍었을 때 글 전체를 보여주던 과거의 방식에서는, 목록 당 5개 이상 나오는거 좀 무리라서요.

그 외의 건의에 대한 답변.

- 게임 제목 자체를 태그에 넣는 사안.
  -> 태그가 너무 어지럽고 복잡해질 수 있음.
  -> 과거 리뷰 / 칼럼 등에 모두 태그 추가하는 건, 굉장한 반복 작업이 필요.

- 태그 검색 창.
  -> 기술적으로 어떻게 해야 할지 모름(...) 민님께 여쭙는다고 답이 나올지도 의문.
  -> 그래도 물어보겠습니다.
  -> 그냥 전체 태그를 모두 보여주는 페이지를, Pig-Min 태그 글에 링크 걸었음.

- 검색 결과의 페이지 인덱스가 리스트 바로 아래도 붙었으면 좋겠음.
  -> 예전에 스킨을 변경하며 시도해보았지만, 1 페이지에 1번만 페이지 인덱스가 출력.
  -> 일단은 '목록'을 클릭했을 때 '글'을 보여주지 않는 방식으로 변경.

이상입니다.

------------------------------------------------------------------------------------------------------

추가 된 건의. (bluedisk)

- 검색 시 페이지 인덱스가 나오지 않음.
  -> 목록 & 글이 모두 출력되는 설정이면, 글 다 나온 끝에 인덱스가 나옴.
  -> 목록만 출력되는 설정이면, 인덱스가 나오지 않음.
  -> 현재 검색 시 목록 & 글이 모두 출력되는 설정으로 재변경.
  -> ... 스크롤이 너무 길어지는 관계로, 어케 해야할지 좀 의논해봐야 하곘습니다.
